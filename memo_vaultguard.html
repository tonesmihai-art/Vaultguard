1 Descriere Generală

VaultGuard este un manager de parole, notițe și carduri de fidelitate — gratuit, open source și fără cloud. Aplicația rulează complet local în browser, fără a transmite date către niciun server extern. Criptarea datelor se realizează cu AES-256, asigurând confidențialitatea exclusiv pe dispozitivul utilizatorului.

Proiectul este implementat ca o aplicație single-file (tot codul CSS, JavaScript și HTML se află într-un singur fișier index.html), ceea ce simplifică distribuția și instalarea. Interfața este proiectată mobile-first, cu lățime maximă de 480px și suport complet pentru instalare ca PWA pe iOS, Android și desktop.

2 Structura Fișierelor

Proiectul conține 8 fișiere organizate în directorul rădăcină și un subdirector pentru iconuri:

Fișier / Director Tip Rol

3 Arhitectură Tehnică
3.1 Aplicație Single-File

Toată logica aplicației — stilizare CSS, interfața HTML și codul JavaScript (inclusiv criptarea) — este înglobată în index.html. Această abordare elimină dependențele externe la runtime și simplifică distribuția: utilizatorul poate partaja sau arhiva întreaga aplicație printr-un singur fișier.

3.2 Progressive Web App (PWA)

Aplicația este configurată ca PWA complet funcțional, cu suport pentru instalare nativă pe orice platformă modernă. Fișierul manifest.json definește identitatea vizuală și comportamentul la instalare (modul standalone, culori, iconuri, orientare portrait). Service Worker-ul (sw.js, versiunea v15) implementează o strategie cache-first pentru toate asset-urile, garantând funcționarea offline după prima încărcare.

3.3 Design Mobile-First

Interfața este construită pentru ecrane mobile, cu lățime maximă de 480px și layout centrat pe desktop. Tema vizuală folosește un gradient întunecat (deep indigo / purple — #0c0e1a → #1a1040), cu accente violet (#6366f1, #a855f7). Animațiile CSS (slideUp, fadeIn, fadeInScale) asigură o experiență fluidă pe touch-screen.

3.4 Securitate & Criptare

Datele utilizatorului sunt criptate cu AES-256. Nu există comunicare cu servere externe — toate operațiunile se desfășoară local, în browser. Modelul de securitate este bazat pe o parolă master introdusă la login, fără recuperare externă.

4 Funcționalități Identificate

Pe baza analizei codului sursă, aplicația include:

  • Manager parole — stocare credențiale cu titlu, utilizator, parolă, URL și categorie, cu căutare și filtrare în timp real.
  • Notițe criptate — pagină dedicată pentru note personale, separate de parole.
  • Carduri de fidelitate — modul pentru stocarea și gestionarea cardurilor de loialitate.
  • Autentificare cu parolă master — ecran de login dedicat cu toggle vizibilitate parolă.
  • Categorii vizuale — sistem de categorii cu iconuri colorate pe grid 4 coloane (Social, Email, Banking etc.).
  • Căutare în timp real — câmp de căutare cu filtrare instantanee a intrărilor.
  • Interfață modală — adăugare / editare intrări prin modal animat tip bottom-sheet (slideUp).
  • Buton FAB — buton flotant de adăugare rapidă, fixat deasupra barei de navigare.
  • Funcționare offline completă — prin Service Worker cu cache v15.
  • Instalare PWA — suport nativ Chrome, Edge, Safari (iOS) și Android.
5 Observații & Recomandări
Prioritate Observație Recomandare
Ridicată Fără teste automate identificate Introducerea unor teste de bază pentru logica de criptare / decriptare
Medie Fișierul index.html este un monolit (>55.000 tokens) Separarea CSS și JS în fișiere distincte îmbunătățește mentenabilitatea
Medie Service Worker la versiunea v15 — actualizări frecvente, fără changelog Documentarea modificărilor într-un fișier CHANGELOG.md
Scăzută readme.txt conține caractere corupte (encoding greșit) Re-salvare cu encoding UTF-8 pentru instrucțiuni lizibile
Scăzută start.bat fără suport non-Windows Adăugarea unui start.sh pentru macOS / Linux
★ Concluzie

VaultGuard este un proiect bine definit, cu arhitectură coerentă și un scop clar: manager de date sensibile, gratuit, offline și open source. Abordarea single-file simplifică distribuția și este potrivită pentru proiectele individuale. Principalele oportunități de îmbunătățire vizează structurarea codului, testarea automată și documentarea mai detaliată.
